Hexo博客搭建流程

发表于2022-05-11|更新于2023-12-25
|阅读量:

摘要

​ 作为一个理工科的学生以及未来计算机行业的从业者，随时记录在日常学习和工作中的灵感和经验是非常有必要的，它们可以帮助自己提升自己的能力和开拓自己的思想，用博文来记录这些是非常方便的，博文可以发布到各大平台上，但是那总归是别人的地盘，我们肯定是想要有一个自己的地盘，以便可以实现对它的高度自定义，比如让界面变得更好看，或者是让内容结构变得更合理，在和别人交流的时候我们也可以互相交换博客的地址来进行互相学习。

​ 我在我这个台式机上搭建了一个hexo博客，但是以后我肯定是会换电脑的，那么换电脑之后我该怎么持续更新博客呢？只能是重新搭建一下相关的环境，这个过程涉及到一些小细节，我把这些步骤和细节写道这篇博文中持续更新，这样换电脑之后就可以很方便的重新搭建相关环境快速进行更新博客了。

​ 注：以下的hexo和git操作都是在存放博客的文件夹下进行操作的。

我想要更换电脑写博客怎么办

  • 这个主要是要先理解github page本质上也是一个repository，因此我们可以使用git来做版本的同步，但是由于之前使用hexo init命令后生成的.gitignore文件中将hexo框架的运行环境等文件设置了提交忽略，这就意味着我们在将github上的项目pull到一个新位置的时候要先在新位置运行hexo init命令以生成hexo需要的环境。
  • 然后我们需要运行git init来将该文件夹设置为git本地仓库，再然后为了防止产生冲突，我们暂时把除了.git外的文件转移到另外一个文件夹中，然后我们运行git pull 路径命令把东西都拉下来，然后我们再将之前转移走的文件转移回来，如果出现冲突则用pull下来的文件。
  • 然后我们可以通过命令hexo clean、hexo g、hexo s来刷新一下hexo的环境并运行，此时我们访问本机的4000端口，发现页面并不能正常显示，会出现一句话：extends includes/layout.pug block content include includes/recent-posts.pug include includes/partial
  • 这个时候我们先停下hexo服务端，然后执行npm install --save hexo-renderer-jade hexo-generator-feed hexo-generator-sitemap hexo-browsersync hexo-generator-archive，然后再执行hexo clean、hexo g，之后启动hexo服务端，发现页面就能正常显示了。
  • 之后就和之前一样用就行了。

我想在博文中显示pdf怎么办

  • 首先安装一个hexo对pdf支持的一个插件：npm install --save hexo-pdf
  • 然后在source文件夹中创建存放pdf的文件夹，source文件夹是引用pdf文件的根目录
  • 然后在博文中使用{% pdf pdf文件路径 %}来引用pdf文件

hexo如何删除博文

  • 如果博客还没有部署到github page上，那么直接删除具体博文后，用hexo clean、hexo g，hexo d即可实现删除并部署博文。
  • 如果博客已经部署到github page上了，那么要删除具体博文之后，再删除本地文件中的.deploy_git文件夹，然后再用hexo clean、hexo g，hexo d即可实现删除并部署博文。
